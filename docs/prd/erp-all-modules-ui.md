## 二、销售模块

### 2.1 报价详情 (ErpQuotationDetail)

**BPF**: `草稿 → 已发送 → 已接受 → 已转订单` / `已拒绝` / `已过期`

**Smart Buttons**: `[📋 订单 {count}]` — 报价转换的订单

**Header Facets**:

| 左列 | 右列 |
|------|------|
| 客户名称（链接） | 小计 |
| 联系人 | 税额 |
| 有效期至（到期倒计时高亮） | 总计 |
| 付款条件 | |

**Tabs**: `[报价明细] [活动日志]`

**报价明细表**:

| 列 | 说明 |
|----|------|
| 产品 | PLM 产品名称 |
| SKU | 可选 |
| 数量 | 可编辑 |
| 单价 | 可编辑 |
| 折扣% | 可编辑 |
| 税率% | |
| 行合计 | 自动计算 |

**操作**:
- draft: [发送报价] [编辑] [删除]
- sent: [标记接受] [标记拒绝] [编辑]
- accepted: [转为订单 ▶]

### 2.2 客户详情 (ErpCustomerDetail)

**不用 BPF**（客户没有线性流程）

**Header Facets**:

| 左列 | 右列 |
|------|------|
| 客户名称 | 信用额度 |
| 类型（企业/个人/经销商） | 应收余额 |
| 税号 | 逾期金额（红色高亮） |
| 联系人 / 电话 / 邮箱 | 状态 |
| 开票地址 | |

**Smart Buttons**: `[📋 订单 {count}] [💰 发票 {count}] [📄 报价 {count}] [🔄 退货 {count}]`

**Tabs**:
- `[订单历史]` — 该客户所有订单，按时间倒序
- `[发票记录]` — 所有发票 + 收款状态

- `[活动日志]`

## 三、库存模块

### 3.1 库存总览 (ErpInventoryDashboard)

**类型**: Analytics Page

**KPI 卡片 (4个)**:
- 库存总额（按物料成本 × 数量汇总）
- 物料种类数
- 低库存预警数
- 平均周转天数

**主视图 — 库存搜索表**:
```
🔍 搜索物料编码/名称...    [仓库: 全部▾]  [状态: 全部▾]

物料编码    物料名称      仓库      数量    预留    可用    成本单价   库存金额    状态
──────────────────────────────────────────────────────────────────────────
PCB-0023   主板          原材料仓   230     50     180    ¥45.00   ¥10,350   🔴 低
IC-0045    芯片          原材料仓   1200    300    900    ¥12.50   ¥15,000   🟢 正常
Lens-01    镜片          半成品仓   0       0      0      ¥85.00   ¥0        🔴 缺货
```

**状态色标**:
- 🟢 正常: 可用量 > 安全库存 × 1.5
- 🟡 注意: 安全库存 < 可用量 ≤ 安全库存 × 1.5
- 🔴 低: 可用量 ≤ 安全库存
- ⚫ 缺货: 可用量 = 0

**点击物料行展开出入库流水**（行展开模式）:
```
▼ PCB-0023 主板
  日期        类型    数量    批次          关联单据        操作人
  04-02 14:30 入库    +500   LOT-0402-01  PO-0089         仓管员
  04-01 10:00 出库    -200   LOT-0315-03  WO-0012         李工
  03-28 09:15 入库    +300   LOT-0328-01  PO-0085         仓管员
```

**底部图表区**:
- 左: 库存金额 Top 10 物料（柱状图）
- 右: 出入库趋势（最近 30 天折线图）

## 四、生产模块

### 4.1 工单详情 (ErpWorkOrderDetail)

**BPF**: `草稿 → 已下达 → 生产中 → 已完工 → 已关闭` / `已取消`

**Smart Buttons**: `[📋 订单 {0/1}] [📦 领料 {count}] [📊 报工 {count}] [🔍 OQC {count}]`

**Header Facets**:

| 左列 | 右列 |
|------|------|
| 产品名称（PLM 链接） | 计划数量 |
| BOM 版本（PLM 链接） | 已完工数量 |
| 来源订单（ERP 链接） | 报废数量 |
| 成品入库仓 | 良率（完工/计划 %） |
| 计划开始 ~ 计划完成 | 实际开始 ~ 实际完成 |

**Tabs**:
- `[BOM 物料]` — BOM 展开的物料列表 + 库存状态 + 已领/待领
  ```
  物料        BOM用量   需求量   已领    待领    库存    状态
  PCB-0023    1        1000    800    200    180    🔴 不足
  IC-0045     2        2000    2000   0      900    🟢 已齐
  ```
- `[领料记录]` — 领料事务列表
- `[报工记录]` — 工序报工：工序名/良品/不良/报废/操作员/时间
- `[OQC 检验]` — 出货检验结果
- `[活动日志]`

**操作**:
- draft: [下达工单 ▶] — 检查物料齐套状态
- released: [领料出库 ▶] [开始生产]
- in_progress: [报工] [完工入库 ▶]
- completed: [创建 OQC] [关闭工单]

**领料操作弹窗** (点击 [领料出库]):
```
┌─ 工单领料 WO-0012 ─────────────────────────┐
│                                             │
│ 物料        需求    已领    本次领料   仓库   │
│ PCB-0023    1000    800    [200  ]   [原材料▾]│
│ IC-0045     2000    2000   [0    ]   [原材料▾]│
│                                             │
│              [取消]  [确认领料 ▶]              │
└─────────────────────────────────────────────┘
```

## 九、统一 Object Page 模板

所有 Detail 页面共享同一个设计模板，只是内容不同：

```
┌──────────────────────────────────────────────────────┐
│ ← 返回列表    {单据编号}              [操作按钮区]    │
│                                                      │
│ ┌─ BPF 状态条 ────────────────────────────────────┐ │
│ │ ● 步骤1 ── ● 步骤2 ── ○ 步骤3 ── ○ 步骤4      │ │
│ │                                 [下一步操作 ▶]  │ │
│ └──────────────────────────────────────────────────┘ │
│                                                      │
│ ┌─ Smart Buttons ─────────────────────────────────┐ │
│ │ [📄 关联1 {n}] [📦 关联2 {n}] [💰 关联3 {n}]    │ │
│ └──────────────────────────────────────────────────┘ │
│                                                      │
│ ┌─ Header Facets ─────────────────────────────────┐ │
│ │ 左列字段组          │  右列字段组                  │ │
│ └──────────────────────────────────────────────────┘ │
│                                                      │
│ ┌─ Tabs ──────────────────────────────────────────┐ │
│ │ [Tab1] [Tab2] [Tab3] [Tab4]                     │ │
│ │ {Tab 内容区}                                     │ │
│ └──────────────────────────────────────────────────┘ │
│                                                      │
│ ┌─ Chatter ───────────────────────────────────────┐ │
│ │ [记录备注...]                                    │ │
│ │ ● 时间  操作人  操作描述                          │ │
│ └──────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘
```

每个 Detail 页面只需要定义:
1. BPF 步骤列表 + 状态映射
2. Smart Buttons 配置（关联实体 + 查询条件）
3. Header Facets 字段
4. Tab 列表 + 每个 Tab 的表格列定义
5. 操作按钮（状态 → 可用操作的映射）
