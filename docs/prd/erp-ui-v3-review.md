# ERP UI v3 迭代 — 基于业务场景的差异化设计

> 版本: v3.0 | 日期: 2026-04-03
> 性质: 对 v2 版全模板化 UI 的深度反思与重构方案

---

## 一、v2 的核心问题

### 问题诊断

v2 用了 2 个模板套了 32 个页面：
- **List 模板**：标题 + 4 个统计卡片 + 搜索框 + Table
- **Detail 模板**：BPF + Smart Buttons + Header Facets + Tabs + Chatter

这种做法的本质错误是：**把 UI 模板当成了产品设计**。

真正的 ERP 设计不是"每个页面长什么样"，而是"每个业务场景的用户在做什么，他需要看到什么，他的操作路径是什么"。不同业务场景的用户需求完全不同，强行统一模板会导致所有页面都"能用但不好用"。

### 对标分析：头部 ERP 怎么做

| 系统 | 做法 | 为什么好用 |
|------|------|-----------|
| **Odoo 17** | 销售用管道看板、库存用即时搜索+操作面板、生产用调度甘特图、财务用专业账簿界面 | 每个模块的 UI 都围绕该模块的核心动作设计 |
| **SAP Fiori** | 销售用 Overview Page、库存用 Worklist、财务用 Analytical List Page、生产用 Monitor 模式 | 5 种 Floorplan 但每种用在对的场景 |
| **NetSuite** | 销售有 Pipeline Dashboard、库存有 Stock Status 页面（红黄绿）、财务有专业 GL 界面 | 每个模块有自己的"杀手功能页面" |
| **Dynamics 365** | 销售有 Deal Room、库存有 Warehouse App、财务有 Financial Insight | 不同角色连入口界面都不一样 |

**共同点：没有任何一个成功的 ERP 系统是"一套模板打天下"的。每个模块都有针对其业务特性设计的独特页面。**

## 二、每个模块的业务本质分析

### 销售模块 — 本质是"推动交易前进"

**用户画像**：销售人员/销售主管
**核心动作**：跟进客户、推动订单、催促回款
**关键场景**：
- "我有哪些客户在跟进？每个到什么阶段了？" → **管道看板**
- "这个客户的历史交易和信用情况？" → **客户 360 视图**
- "哪些订单该催发货了？哪些该催回款了？" → **行动驱动列表**
- "这个月业绩怎么样？" → **销售分析仪表盘**

### 库存模块 — 本质是"物的位置和数量"

**用户画像**：仓管员/物流
**核心动作**：入库、出库、查库存、盘点
**关键场景**：
- "这个物料在哪？有多少？" → **即时搜索 + 库位可视化**
- "今天有什么要入库/出库？" → **操作待办面板**
- "哪些物料快没了？" → **预警看板**
- "这个序列号的全生命周期？" → **追溯时间线**

### 生产模块 — 本质是"计划和执行的闭环"

**用户画像**：生产计划员/车间主管
**核心动作**：排产、跟踪进度、处理异常
**关键场景**：
- "我需要采购什么？需要生产什么？" → **MRP 向导（已有）**
- "车间现在在做什么？进度如何？" → **产线看板/甘特图**
- "这批产品的良率怎么样？" → **良率趋势图**
- "这个工单缺什么料？" → **BOM 齐套检查**

### 财务模块 — 本质是"数字的精确记录和分析"

**用户画像**：会计/财务主管
**核心动作**：录凭证、对账、出报表
**关键场景**：
- "这笔收入的凭证怎么做？" → **凭证录入（专业会计界面）**
- "应收账款的账龄？" → **账龄分析图**
- "这个月利润多少？" → **财务报表（专业格式）**
- "哪些发票还没收到钱？" → **应收催收工作台**

### 质量模块 — 本质是"发现问题、解决问题、防止再发"

**用户画像**：质量工程师
**核心动作**：检验、记录不良、分析根因、跟踪改善
**关键场景**：
- "这批货的检验结果？" → **检验记录卡**
- "最近的质量趋势？" → **SPC 控制图 + Pareto 图**
- "这个问题的根因和改善进展？" → **8D/CAPA 跟踪看板**

## 六、设计原则总结

| 原则 | v2 的做法 | v3 的做法 |
|------|----------|----------|
| 页面设计出发点 | 数据实体 | 用户的工作场景 |
| 页面结构 | 统一模板 | 每个场景独特设计 |
| 信息组织 | 字段平铺 | 按业务逻辑分层 |
| 操作方式 | 进入详情页再操作 | 在列表/看板上直接操作 |
| 数据展示 | 表格为主 | 图表+看板+时间线+搜索多种形态 |
| 设计判断依据 | "这个实体有什么字段" | "这个角色在做什么决策" |
