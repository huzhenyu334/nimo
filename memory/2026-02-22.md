## Agent进化架构设计 v1.0（重大设计讨论）
泽斌和我深入讨论了agent进化的完整架构，沉淀到 `docs/agent-evolution-architecture.md`

### 核心框架：Agent = 认知(Prompt) × 知识(Knowledge) × 工具(Tools)

**认知体系**：模板继承链 BaseAgent → 角色模板 → 实例
**知识体系**：四层scope（全局→角色类型→流程→Step），系统驱动升级
**工具体系**：角色工具集继承，所有工具MCP化

### 关键洞察（泽斌原话）
1. "agent能使用什么工具，决定了他的能力边界，就像人需要软件一样"
2. "没有的工具就自己造" → 飞轮3：工具进化闭环
3. "有了这个飞轮，业务就能自己高效的运转起来，完成自我进化"
4. 知识继承不只是三层，而是多维度：全局通用/角色类型通用/流程通用/step通用

### 三大飞轮
1. 任务执行闭环（已跑通）
2. 知识进化闭环（建设中）
3. 工具进化闭环（未来方向）

### Plugin vs MCP 决策
- 初步结论：Plugin = Channel，MCP = Tool
- 看了OpenClaw Plugin文档后修正：Plugin还适合background service、Gateway HTTP扩展、Provider auth
- ACP可能是混合模式：核心调度Plugin + tool暴露MCP
- **新工具开发策略：默认MCP，除非需要进程内能力**

## 深入OpenClaw架构讨论（与泽斌）
- Skills: SKILL.md格式、per-agent vs shared、加载优先级（workspace > ~/.openclaw/skills > bundled）、零配置设计
- Tools三层：Built-in核心 → Plugin注册 → MCP Server
- 文件加载机制：自动注入（AGENTS/SOUL/USER/IDENTITY/TOOLS/MEMORY）vs 搜索加载（bank/shared/历史日志）vs 按需读取（skills/）
- 工具进化层级L0-L4：无工具→exec→Skill+exec→TypedTool/MCP→ACP编排

- Skill vs ACP编排：软编排（灵活低成本）vs 硬编排（可靠可审计）
- Skills价值判断：通用知识skills贬值（模型越强越不需要），私有知识skills永久价值
- per-agent tool profiles: minimal/coding/messaging/full
