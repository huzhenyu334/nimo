# ACP 流程设计规范（速查版）

完整版在知识库，这里是设计流程时的快速检查清单。

## 设计六步法

1. **画人工版**：如果全靠人做，怎么做？
2. **识别执行者**：每步是系统查询、AI Agent、还是人？→ 选executor
3. **定义输入输出**：每步需要什么、产出什么？画数据流
4. **设计异常路径**：每步失败怎么办？
5. **确定步骤粒度**：单一职责、可观测、可重试
6. **写YAML**：前5步完成后才动手

## 命名规范

- **流程名**：`[业务域]-[动作]`，中文，如"软件项目立项"
- **步骤ID**：`[动词]_[名词]`，英文下划线，如 `check_project`、`generate_blueprint`
- **步骤Name**：中文简述，如"检查项目是否存在"

## 步骤粒度检查

- [ ] 每步只做一件事
- [ ] 看ID就知道干什么
- [ ] 失败后重试安全吗

## Agent Prompt模板

```
你是[角色]。请对[对象]执行[任务]。

## 上下文
{{steps.上游.output.data/text}}

## 任务
1. 具体任务一
2. 具体任务二

## 输出
[output:key=value]
```

## 人工步骤规则

- 必须提供**足够上下文**让人决策（不要让人再去查别的系统）
- 审批步骤必须有 boolean 字段（approved/passed）
- 始终提供 notes/reason 文本字段

## 常用模式

| 模式 | 结构 |
|------|------|
| 查询→判断→分支 | nocodb → switch → branch_a / branch_b |
| Agent链 | agent₁ → agent₂（≤3层） |
| 生成→注册双写 | agent(创建文档) → nocodb(注册索引) |
| 人工审批关卡 | auto_work → human → if gate → proceed/rollback |
| 子流程调用 | subprocess(已有流程) |

## 错误处理速查

| on_failure | 用途 |
|------------|------|
| abort | 失败影响全局 |
| skip | 增强型步骤，跳过不影响 |
| pause | 需人工排查 |
